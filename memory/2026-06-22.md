## 记忆系统架构讨论

用户要求详细了解 OpenClaw 记忆系统的设计与工作机制，讨论涵盖以下内容：

### 核心架构
- 三层记忆结构：`MEMORY.md`（长期）、`memory/YYYY-MM-DD.md`（每日笔记）、`DREAMS.md`（梦境日记）
- 核心理念：纯文本文件即记忆，无隐藏状态
- 回忆工具：`memory_search`（语义+关键词混合检索）、`memory_get`（精确读取）

### 整理精炼机制（四种）
1. **自动记忆刷新（Memory Flush）**：压缩前静默触发，提醒模型保存重要上下文
2. **心跳记忆维护**：定期在心跳中审阅每日笔记，提炼到 MEMORY.md
3. **做梦系统（Dreaming）**：三阶段后台整合（Light → REM → Deep），仅 Deep 阶段写入 MEMORY.md
4. **推断承诺（Commitments）**：短期跟踪机制，从对话中推断未来跟进点

### 晋升条件（Deep Phase）
- 六维加权评分：频率(0.24)、相关性(0.30)、查询多样性(0.15)、时效性(0.15)、巩固度(0.10)、概念丰富度(0.06)
- 三道硬性门槛：最低分数、最低召回次数、最低唯一查询数

## 系统激活
- 做梦系统：通过 `openclaw config set` 启用，每天凌晨 3 点扫描
- 心跳：通过 `openclaw config set` 启用，每 30 分钟一次
- Gateway 重启后配置生效
